### 1. Spectator Cleanup Bug

**Issue:** Spectators were not removed from `state.players` when leaving, causing subsequent tests to fail.

**Impact:** Spectator test scenarios now clean up correctly; no stale spectator state persists.

### 2. Reconnection Test Logic

**Issue:** Reconnection tests assumed host always plays Black. In multiplayer scenarios with randomized player order, this assumption broke.

**Solution:** Implemented `playMoveForCurrentTurn()` helper function that:
- Queries current turn from game state
- Maps player ID to playerIndex
- Selects valid move for that player's pieces (not hardcoded Black)
- Eliminates order-dependent test logic

**Impact:** Reconnection tests work across all player configurations; patterns established for future multiplayer E2E work.

### 3. Backgammon Timeout

**Issue:** Backgammon win simulation exceeded 30s Playwright default timeout.

**Fix:** Set explicit 180s timeout for backgammon win scenarios.

**Impact:** Win path coverage now complete for Backgammon.

## Key Learning

The `playMoveForCurrentTurn` pattern is reusable for any multiplayer E2E scenario where move selection depends on current game state rather than static player assignment. Document this in game plugin patterns for future implementors.
