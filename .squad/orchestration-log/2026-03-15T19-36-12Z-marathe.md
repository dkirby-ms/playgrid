## Trigger

User reported Azure Container Apps health check probe failure after UAT deployment.

## Investigation Scope

- `infra/main.bicep` — infrastructure-as-code output variables
- `server/src/index.ts`, `server/src/db.ts`, `server/src/config.ts` — runtime startup, DB initialization, env config
- `Dockerfile` — container image build steps
- `.github/workflows/deploy-uat.yml`, `.github/workflows/deploy-prod.yml` — deployment automation

## Root Cause Found

**Duplicate `https://` protocol prefix** in workflow health check URL construction:

- `CONTAINER_APP_FQDN` environment variable from Azure **already includes** `https://` protocol
  - Example: `https://playgrid-uat.orangedune-0437f62b.centralus.azurecontainerapps.io`
- Workflows were prepending another `https://` unconditionally
- Result: Malformed health check URL `https://https://playgrid-uat...` → immediate probe failure

## Files Modified

1. `.github/workflows/deploy-uat.yml`
   - Removed protocol prefix prepend from health check URL construction
   - Uses `CONTAINER_APP_FQDN` directly as-is

2. `.github/workflows/deploy-prod.yml`
   - Same fix applied for production parity

## Decision Made

**GitHub environment variables for URLs MUST include protocol prefix. Workflows use them directly without transformation.**

Documented in `.squad/decisions/inbox/marathe-healthcheck-fix.md`
