### Design Decisions

The CPU framework in BaseGameRoom now uses game-specific executor methods rather than a single polymorphic interface. This allows each game to handle multi-step turns (Backgammon: roll → move × N) without forcing a unified interface.

Future games simply need to:
1. Create a `CpuOpponent.ts` module
2. Add gameType to `cpuOpponentEnabled` gate
3. Implement an `execute{Game}CpuTurn()` method
