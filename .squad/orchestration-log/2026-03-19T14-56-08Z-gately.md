## Work Summary

Fixed two critical Dominos bugs affecting game initialization and reconnection.

### Bugs Fixed

1. **Board appears non-functional on game start** — Player hand tiles missing on initial join.
2. **Hand tiles invisible after page-refresh reconnect** — Tiles lost on reconnection.

### Root Cause

Race condition in Colyseus SDK message delivery. Server sends `player-data` via `client.send()` in `onJoin()` before the client has registered its `room.onMessage("player-data")` handler. Unhandled messages are silently dropped; no replay or buffering occurs. Player hand array stays empty.

Secondary: async `SceneManager.transitionTo()` creates window where state patches arrive before `onStateChange` handler registered.

### Solution (3 files, 18 lines)

1. **BaseGameRoom** — Added `"request-player-data"` message handler for on-demand resend.
2. **DominosRenderer** — After handler registration, sends `"request-player-data"` request to guarantee hand delivery.
3. **GameScene** — After registering `onStateChange`, immediately invokes handler with current state to flush pending patches.

### Key Learning

Never rely on server-initiated messages during `onJoin()`. The Colyseus handshake completes and delivers messages before the renderer has registered game-specific handlers. Use server-push paired with client-pull for critical state.
