## Work Summary

Fixed Risk setup phase deadlock by adding global `allDone` check in `placeArmy` action handler to auto-transition from setup/placing phase to playing/reinforce phase when all players have 0 armies remaining.

## Context

Bug was reported LIVE on UAT by dkirby-ms. The game deadlocked in setup phase because the transition check only considered the current player's state. After the last player placed armies, the turn wrapped to an already-finished player who had no valid moves to trigger `endPhase`.

## Decision

When a game plugin's action completes a per-player requirement during a round-robin phase, the action itself must check the global completion condition and trigger the phase transition. Do not rely on separate explicit actions.
