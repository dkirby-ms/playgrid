# Orchestration Log: Gately — Drag Disappearing Bug Fix

**Date:** 2026-03-19T14:23:20Z  
**Agent:** Gately (Gameplay & Interaction)  
**Focus:** Fix drag-and-drop flakiness (pieces disappearing when dragged slowly)

## Problem Statement

Drag-and-drop was unreliable in multiplayer games. When players dragged pieces slowly (building toward the 6px promotion threshold), pieces would disappear from under their cursor mid-drag. Root cause: both `CheckersRenderer` and `DominosRenderer` unconditionally cancelled drags on every Colyseus state sync.

## Root Cause Analysis

- Colyseus state syncs arrive frequently (timer ticks, opponent moves, etc.)
- `onStateChange()` called `dragHelper.cancel()` unconditionally
- Active drags were destroyed even if the piece and tile were still valid
- Result: Promoted drag proxy vanished, piece no longer followed cursor

## Solution

1. **Renderers** — Modified `onStateChange()` to cancel drags only when the source is invalidated:
   - Piece was captured (removed from board)
   - Tile left the hand (for Dominos)
   - Turn changed (not applicable for current renderers)

2. **DragHelper.cancel()** — Now always calls `onDragCancel` callback for both promoted and pending drags; previously silent for pending drags, leaving renderer state stale.

3. **Hover guard** — `handleSquareHover()` in Checkers now blocks during pending drags (not just promoted ones), preventing unnecessary redraws during the 6px build-up.

## Results

- All games using DragHelper benefit (Checkers, Dominos currently)
- Existing click-vs-drag distinction (6px threshold) unchanged
- 4 new regression tests added to `DragHelper.test.ts`
- 583 total tests passing
- 0 regressions in existing tests
