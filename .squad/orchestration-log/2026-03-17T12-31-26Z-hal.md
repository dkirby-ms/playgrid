## Summary

Coordinated investigation of 15 failing E2E tests across Checkers, Risk, Backgammon, spectator scenarios, and reconnection flows. Identified root causes and delegated implementation work to Pemulis, Steeply, and Coordinator.

## Identified Issues

1. **E2E Snapshot Extraction (6 specs failing)**
   - Caused by Phase 4 sidebar redesign breaking `playerColorText` and `statusText` selectors
   - Solution: Fallback chain for snapshot extraction across all 6 specs
   - Assignee: Coordinator

2. **Risk Reinforcement Bug**
   - Reinforcements calculated for wrong player during turn transition
   - Root cause: `endPhase` computed reinforcements before `endsTurn: true`
   - Solution: New `onTurnStarted` lifecycle hook in IGamePlugin
   - Assignee: Pemulis

3. **CPU Opponent Detection**
   - Spectator tests failing due to improper CPU opponent identification
   - Root cause: Schema-level boolean field not synced reliably
   - Solution: Use session ID (`"cpu-opponent"`) instead of schema field
   - Assignee: Coordinator

4. **Spectator Cleanup**
   - Spectators not removed from state.players on leave
   - Solution: Delete spectators from state.players in onPlayerLeave
   - Assignee: Steeply

5. **Reconnection Test Logic**
   - Tests assuming host always plays Black; broke in multiplayer scenarios
   - Solution: `playMoveForCurrentTurn` helper using playerIndex, not hardcoded Black
   - Assignee: Steeply

6. **Backgammon Timeout**
   - Win simulation timing out with default 30s Playwright timeout
   - Solution: 180s timeout for backgammon win scenarios
   - Assignee: Steeply

## Result

All 40 E2E tests passing, 292 unit tests passing, lint clean. No regressions.
