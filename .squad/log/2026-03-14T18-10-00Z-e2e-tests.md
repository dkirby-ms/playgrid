## Decisions Made

### 1. Dedicated Lobby Playwright Config

**Decision:** Use `playwright.lobby.config.ts` for the lobby E2E suite; root `playwright.config.ts` re-exports it.

**Why:**
- Isolates lobby tests from unrelated browser specs (e.g., older experimental tests)
- Allows gating on stable lobby workflow without blocking other tests
- Conventional Playwright setup for future test suites

**Operational:**
- Base URL: `http://127.0.0.1:3000`
- Server: `DATABASE_URL= npm run dev`
- Browser: Chromium only
- Workers: 1
- Match: `**/lobby.spec.ts`

---

### 2. Grey Box E2E Pattern for Game Plugins

**Decision:** All future game plugins must use Grey Box E2E (UI via DOM, moves via harness, assertions on state).

**Why:**
- Canvas rendering is environment-dependent and flaky
- Game logic and state synchronization are what matter for correctness
- Avoids brittle pixel-level assertions and coordinate-based clicks
- Deterministic and reproducible across all environments

**Pattern:**
1. Expose `window.__PLAYGRID_E2E__.app.gameRoom` when `?e2e=1`
2. Use Playwright for lobby/waiting-room interactions
3. Send game moves through the browser `gameRoom` object (real player connection)
4. Assert on synchronized server state
5. Use deterministic move sequences (e.g., 31 moves for Checkers)

**Applies To:** Checkers (done), and all future plugins (Backgammon, Dominoes, Poker, Hearts/Spades, Chess, Risk)

---

### 3. PR Review Gate (Stacked Branches)

**Decision:** PRs targeting `dev` must be independently reviewable against the base branch.

**Rule:**
- No unrelated commits from other open PRs in the diff
- No cross-agent history or test changes unless directly required by the issue
- If a branch is stacked on another unmerged PR, rebase/cherry-pick or refresh after the lower PR lands before approval

**Why:**
- Keeps review scope explicit
- Prevents accidental double-approval of unrelated work
- Makes rollback/revert boundaries match the issue being solved
- Critical with multiple agents landing adjacent work to the same branch

**Example:** PR #57 was initially blocked because it included unrelated commits from PR #56. After rebasing, it was approved.
